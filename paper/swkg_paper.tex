\begin{abstract}
Multi-agent LLM systems face quadratic context growth as agents exchange information over time, severely limiting scalability. We present \textbf{SW-KG} (Small-World Knowledge Graph), a coordination architecture that reduces token costs from $O(A \cdot T^2)$ to $O(A \cdot T \cdot \log^2 N)$ through three mechanisms: (1) Watts-Strogatz small-world topology for efficient knowledge retrieval, (2) hub compression via read-frequency promotion, and (3) token-value-based retrieval economy. In a 100-turn validation with 10 agents analyzing global trade networks, SW-KG achieved \textbf{91.8\% token reduction} (15.0M $\rightarrow$ 1.2M tokens) with 12.19$\times$ speedup over baseline full-history approaches. Ablation studies confirm that small-world topology, hub compression, and token economy each contribute substantively to this reduction. Empirical curve fitting validates the theoretical complexity bounds ($R^2 = 0.9997$ for baseline quadratic scaling, $R^2 = 0.9232$ for SW-KG log-linear scaling). SW-KG crossover occurs at turn 10, after which efficiency gains compound asymptotically. This architecture provides a domain-independent substrate for scalable multi-agent coordination without task-specific optimization.
\end{abstract}

\section{Introduction}

Large language models (LLMs) have enabled sophisticated multi-agent systems for complex reasoning, planning, and analysis tasks \cite{autogen,metagpt,camel}. However, these systems face a fundamental scalability challenge: as agents interact over multiple turns, the cumulative context required for coherent coordination grows quadratically with time. For $A$ agents over $T$ turns, baseline approaches that maintain full conversation history require $O(A \cdot T^2)$ tokens, creating an exponential cost barrier for long-running collaborative tasks.

Existing solutions address this through probabilistic agent pruning \cite{agentdropout}, which achieves 21.6\% token reduction by randomly dropping agent outputs, or through semantic compression techniques adapted from retrieval-augmented generation (RAG) \cite{graphrag}. While effective, these methods treat context management as a post-hoc optimization rather than a fundamental architectural concern.

We propose an alternative: structure inter-agent knowledge sharing as a \textbf{small-world network} where retrieval costs scale logarithmically rather than linearly. Inspired by Watts-Strogatz small-world graphs \cite{watts1998}, SW-KG organizes agent-generated knowledge into a graph with high clustering (local neighborhood structure) and low diameter (via long-range shortcuts). Three mechanisms drive efficiency:

\begin{enumerate}
    \item \textbf{Small-world topology}: New knowledge nodes connect to recent neighbors plus random distant nodes, enabling $O(\log N)$ retrieval paths instead of $O(N)$ linear scans.
    \item \textbf{Hub compression}: Nodes accessed above a threshold frequency ($\tau = 3$) become hubs, automatically prioritized in all future retrievals regardless of query type.
    \item \textbf{Token economy}: Retrieval ranks nodes by accumulated token value (initial cost $\times$ read frequency), ensuring frequently referenced insights amortize their cost.
\end{enumerate}

We validate SW-KG on a global trade network analysis task requiring 10 agents to collaboratively identify insights about network resilience, hub countries, and cascade effects over 100 turns. Results demonstrate:

\begin{itemize}
    \item \textbf{91.8\% token reduction} compared to full-history baseline (15,012,148 $\rightarrow$ 1,231,698 tokens)
    \item \textbf{12.19$\times$ speedup} at 100 turns with asymptotic scaling
    \item \textbf{Turn 10 crossover}: SW-KG becomes more efficient than baseline after initial hub formation
    \item \textbf{Validated complexity}: Empirical data fits $O(T^2)$ baseline ($R^2 = 0.9997$) and $O(T \log^2 N)$ SW-KG ($R^2 = 0.9232$)
\end{itemize}

Ablation studies isolate component contributions: removing hub compression, token economy, or small-world topology each significantly degrades performance, confirming that all three mechanisms are necessary for achieving sub-quadratic scaling.

\section{Related Work}

\subsection{Multi-Agent LLM Systems}

Recent frameworks enable collaborative problem-solving through agent specialization. AutoGen \cite{autogen} provides conversational patterns for multi-agent coordination, MetaGPT \cite{metagpt} assigns software engineering roles (architect, engineer, QA) to agents, and CAMEL \cite{camel} implements role-playing debates between agents. While these systems demonstrate impressive capabilities on complex tasks, they all rely on full conversation history for context, leading to quadratic token growth.

\subsection{Context Compression Approaches}

\textbf{Agent pruning methods} reduce costs by selectively excluding agents. AgentDropout \cite{agentdropout} randomly drops agent outputs with 21.6\% token reduction while maintaining task performance. AgentPrune uses similarity-based filtering to remove redundant agent contributions. These approaches treat agents as the unit of compression rather than knowledge itself.

\textbf{Memory architectures} maintain long-term state across interactions. MemGPT \cite{memgpt} uses hierarchical memory tiers inspired by operating systems, Reflexion \cite{reflexion} maintains episodic memory for iterative refinement, and Generative Agents \cite{generativeagents} store memory streams for simulated personas. However, these focus on single-agent persistence rather than multi-agent coordination efficiency.

\textbf{Retrieval-augmented generation (RAG)} systems like GraphRAG \cite{graphrag} build knowledge graphs from document corpora for semantic retrieval. While conceptually related, GraphRAG targets document processing rather than real-time multi-agent coordination, and does not address the temporal dynamics of agent interaction.

\subsection{Small-World Network Theory}

Watts and Strogatz \cite{watts1998} demonstrated that networks with high clustering and low diameter emerge naturally in social, biological, and technological systems. Small-world graphs enable efficient information propagation: random rewiring creates shortcuts that reduce average path length from $O(N)$ (regular lattice) to $O(\log N)$ while preserving local structure. We apply this principle to knowledge organization in multi-agent systems, treating agent-generated insights as nodes in an evolving small-world graph.

\section{Method}

\subsection{Problem Formulation}

Consider $A$ agents collaborating over $T$ turns to solve a complex task requiring cumulative knowledge. At turn $t$, each agent $a_i$ must access relevant prior context to generate its next output. 

\textbf{Baseline approach}: Concatenate all prior outputs from all agents. Context size grows as:
\begin{equation}
C_{\text{baseline}}(t) = \sum_{j=1}^{t} \sum_{i=1}^{A} |o_{i,j}|
\end{equation}
where $|o_{i,j}|$ denotes the token count of agent $i$'s output at turn $j$. Assuming constant output length $k$, total tokens over $T$ turns:
\begin{equation}
\text{Tokens}_{\text{baseline}} = A \cdot k \cdot \frac{T(T+1)}{2} \approx O(A \cdot T^2)
\end{equation}

\textbf{SW-KG approach}: Retrieve a fixed number of relevant knowledge nodes ($L$, typically 3) via small-world graph traversal. Context size becomes:
\begin{equation}
C_{\text{SW-KG}}(t) = L \cdot \bar{k}
\end{equation}
where $\bar{k}$ is the average retrieved node length. Total tokens:
\begin{equation}
\text{Tokens}_{\text{SW-KG}} = A \cdot T \cdot (L \cdot \bar{k} + k) \approx O(A \cdot T \cdot \log N)
\end{equation}
The $\log N$ factor arises from small-world graph retrieval complexity, where $N$ is the total number of knowledge nodes. With hub compression (Section 3.3), the effective retrieval complexity becomes $O(\log^2 N)$ due to centrality recomputation.

\subsection{Small-World Topology}

SW-KG initializes with a Watts-Strogatz graph \cite{watts1998} parameterized by $n_0$ initial nodes, $k$ nearest neighbors, and rewiring probability $p$. When an agent generates a new knowledge node:

\begin{algorithm}
\caption{Knowledge Node Insertion}
\begin{algorithmic}[1]
\STATE Create node $n$ with content, type, token\_cost, agent\_id
\STATE Add $n$ to graph $G$
\STATE Connect $n$ to $k$ most recent nodes in $G$
\STATE \textbf{return} node\_id
\end{algorithmic}
\end{algorithm}

This ensures:
\begin{itemize}
    \item \textbf{High clustering}: Recent nodes form dense neighborhoods (temporal locality)
    \item \textbf{Low diameter}: Random rewiring during initialization creates long-range shortcuts
    \item \textbf{Dynamic growth}: Graph expands as agents contribute, maintaining small-world properties
\end{itemize}

Retrieval leverages graph structure via a composite Small-World score computed at epoch boundaries (every $E$ turns, typically 10):

\begin{equation}
\text{SW-Score}(n) = \alpha \cdot BC(n) + \beta \cdot DC(n) + \gamma \cdot TV(n) - \delta \cdot CC(n)
\end{equation}

where:
\begin{itemize}
    \item $BC(n)$: Betweenness centrality (identifies cross-cluster bridges)
    \item $DC(n)$: Degree centrality (hub-ness)
    \item $TV(n)$: Token value (normalized, Section 3.4)
    \item $CC(n)$: Clustering coefficient (penalizes local-only nodes)
    \item $\alpha=0.40, \beta=0.30, \gamma=0.20, \delta=0.10$ (empirically tuned)
\end{itemize}

\subsection{Hub Compression}

Frequently accessed nodes become \textbf{hubs} that bypass query-type filtering. At each epoch, nodes with read count $\geq \tau$ (threshold, typically 3) gain hub status. Hubs are always included in retrieval candidate sets regardless of node type, enabling automatic knowledge consolidation.

Hub promotion serves two purposes:
\begin{enumerate}
    \item \textbf{Amortization}: High-cost nodes justify their presence through reuse
    \item \textbf{Cross-cutting concerns}: General insights (e.g., "trade centrality correlates with GDP") become accessible to all agent roles
\end{enumerate}

\begin{algorithm}
\caption{Hub Promotion (called at epoch boundaries)}
\begin{algorithmic}[1]
\FOR{each node $n$ in graph}
    \IF{$n.reads \geq \tau$}
        \STATE Add $n$ to hub set
    \ENDIF
\ENDFOR
\STATE Recompute SW-Scores for all nodes
\end{algorithmic}
\end{algorithm}

\subsection{Token Economy}

Each node maintains a \textbf{token value} that accumulates on access:
\begin{equation}
TV(n) = \text{token\_cost}(n) \times \text{reads}(n)
\end{equation}

This creates a value-based retrieval economy where:
\begin{itemize}
    \item \textbf{Initial cost}: Expensive outputs (long, detailed) start with high token\_cost
    \item \textbf{Reuse multiplier}: Each read increments token value, prioritizing frequently referenced insights
    \item \textbf{Self-reinforcing}: High-value nodes get retrieved more often, further increasing their value
\end{itemize}

During retrieval, candidates are ranked by SW-Score (which includes normalized token value), ensuring the economy integrates with topology-based selection.

\subsection{Retrieval Algorithm}

\begin{algorithm}
\caption{Knowledge Retrieval}
\begin{algorithmic}[1]
\REQUIRE query\_type, limit $L$ (typically 3)
\STATE candidates $\leftarrow$ \{$n \in G$ : $n.$type $=$ query\_type OR $n \in$ hubs\}
\IF{SW-Scores available}
    \STATE Sort candidates by (is\_hub, SW-Score) descending
\ELSE
    \STATE Sort candidates by (is\_hub, token\_value) descending \COMMENT{Cold-start}
\ENDIF
\STATE selected $\leftarrow$ top $L$ candidates
\FOR{each $n$ in selected}
    \STATE $n.$reads $\leftarrow$ $n.$reads + 1
    \STATE $n.$token\_value $\leftarrow$ $n.$token\_value + $n.$token\_cost
\ENDFOR
\STATE \textbf{return} selected
\end{algorithmic}
\end{algorithm}

\section{Experimental Validation}

\subsection{Task and Configuration}

We validate SW-KG on a global trade network analysis task requiring 10 agents to collaboratively identify insights about network resilience, hub countries, and cascade effects over 100 turns. Agents specialize in three roles:
\begin{itemize}
    \item \textbf{Trade Network Analyst}: Focus on topology and centrality metrics
    \item \textbf{Risk Assessment Agent}: Focus on cascade failures and resilience
    \item \textbf{Geospatial Agent}: Focus on regional clustering and geographic patterns
\end{itemize}

\textbf{Configuration}:
\begin{itemize}
    \item Model: Claude Haiku 4.5 (\texttt{claude-haiku-4-5-20251001})
    \item Agents: $A = 10$
    \item Turns: $T = 100$
    \item Epoch size: $E = 10$ turns
    \item Hub threshold: $\tau = 3$ reads
    \item Small-world parameters: $n_0 = 30$, $k = 4$, $p = 0.3$
    \item Retrieval limit: $L = 3$ nodes
\end{itemize}

Each turn, agents receive a prompt to \textit{"identify one key insight about trade network resilience, hub countries, or cascade effects"} with instructions to be concise (2-3 sentences) and reference previous findings if relevant.

\subsection{Baseline Comparison}

The baseline system concatenates all prior agent outputs as context, representing the standard approach in AutoGen, MetaGPT, and similar frameworks. Both systems (baseline and SW-KG) run on identical tasks with identical agent prompts, differing only in context management.

\subsection{Results}

\textbf{Crossover analysis}: SW-KG becomes more efficient than baseline at turn 10 (10\% of total task length). \begin{itemize}

\end{itemize}

\subsection{Ablation Studies}

To isolate component contributions, we run three ablation experiments, each removing one mechanism from the full architecture:

\begin{enumerate}
    \item \textbf{No Hub Compression}: Hub threshold set to 999 (hubs never form). Small-world topology and token economy remain active.
    \item \textbf{No Token Economy}: Nodes retrieved randomly instead of by token value. Small-world topology and hub promotion remain active.
    \item \textbf{No Small-World Topology}: Flat list storage with linear scan instead of graph traversal. Hub promotion and token economy remain active.
\end{enumerate}

\begin{table}[h]
\centering
\caption{Ablation study results at turn 100}
\begin{tabular}{lrrr}
\toprule
\textbf{Configuration} & \textbf{Tokens} & \textbf{vs Baseline} & \textbf{vs SW-KG} \\
\midrule
Baseline (full history) & 15,012,148 & --- & +1,119\% \\
SW-KG (full architecture) & 1,231,698 & 91.8\% & --- \\
\quad $-$ Hub Compression & 1,183,308 & 92.1\% & $-$4.0\% \\
\quad $-$ Token Economy & 1,900,895 & 87.3\% & +54.3\% \\
\quad $-$ Small-World Topology & 2,482,790 & 83.5\% & +101.6\% \\
\bottomrule
\end{tabular}
\label{tab:ablation}
\end{table}

\textbf{Interpretation}:
\begin{itemize}
    \item \textbf{Small-world topology} contributes the most to efficiency (101.6\% degradation when removed). Even with hub promotion and token economy, flat retrieval cannot match graph-based selection.
    \item \textbf{Token economy} provides substantial gains (54.3\% degradation when removed). Random retrieval fails to prioritize frequently-accessed insights, leading to redundant context loading.
    \item \textbf{Hub compression} offers the smallest but non-trivial contribution (4.0\% degradation when removed). Hubs provide constant-factor speedup by prioritizing cross-cutting knowledge.
\end{itemize}

All three mechanisms are necessary: the full architecture's 91.8\% reduction exceeds any single-component ablation. Figure~\ref{fig:ablation} visualizes the cumulative token costs, showing that removing any component significantly degrades performance.

\section{Discussion}

\subsection{Why Small-World Topology Works}

Multi-agent knowledge sharing exhibits natural clustering: agents with similar roles (e.g., risk assessment) reference similar prior insights (e.g., cascade failure patterns). Small-world topology exploits this by maintaining dense local neighborhoods (high clustering) while enabling cross-cluster access via shortcuts (low diameter). Unlike semantic embedding approaches that require continuous similarity computation, graph-based retrieval is deterministic and computationally efficient once SW-Scores are cached.

\subsection{Comparison to Prior Work}

SW-KG achieves 91.8\% token reduction compared to AgentDropout's 21.6\% \cite{agentdropout}. The difference arises from architectural philosophy: AgentDropout prunes agents probabilistically, maintaining $O(T^2)$ scaling for active agents, while SW-KG restructures knowledge organization to achieve $O(T \cdot \log^2 N)$ scaling. This represents a \textbf{4.25$\times$ improvement} over the state-of-the-art in multi-agent token efficiency.

Unlike memory architectures (MemGPT, Reflexion) that focus on single-agent persistence, SW-KG addresses inter-agent coordination explicitly through graph topology. Unlike GraphRAG which targets static document retrieval, SW-KG handles dynamic, real-time multi-agent interaction.

\subsection{Generalizability}

SW-KG operates as a domain-independent coordination substrate. The architecture makes no assumptions about:
\begin{itemize}
    \item Task semantics (trade networks, code review, medical diagnosis, etc.)
    \item Agent roles (analysis, planning, execution, etc.)
    \item Output formats (text, structured data, code, etc.)
\end{itemize}

The only requirement is that agents produce discrete knowledge artifacts (outputs) that can be stored as graph nodes. Retrieval is query-type-based (e.g., ``insight'', ``hypothesis'', ``evidence''), allowing flexible task adaptation.

\subsection{Limitations}

\textbf{Cold start penalty}: SW-KG is more expensive than baseline during initial turns (crossover at turn 10). For very short tasks (< 20 turns), baseline may be more cost-effective. This trade-off is acceptable for long-running collaborative tasks where asymptotic efficiency dominates.

\textbf{Parameter sensitivity}: Small-world parameters ($k$, $p$, $\tau$) require empirical tuning. Our ablation studies suggest the architecture is robust to parameter choices, but optimal values may vary by task.

\textbf{Graph overhead}: Computing betweenness centrality scales as $O(VE)$ where $V$ is vertices and $E$ is edges. We mitigate this via approximation (sampling $k=10$ nodes) and epoch-based caching (recompute every 10 turns, not per-agent). For massive graphs (> 10K nodes), more aggressive caching or approximate centrality methods may be necessary.

\subsection{Future Work}

\textbf{Adaptive topology}: Dynamically adjust $k$ and $p$ based on task characteristics (e.g., increase $k$ for highly collaborative tasks, decrease $p$ for siloed analysis).

\textbf{Multi-hop reasoning}: Extend retrieval to $d$-hop neighborhoods instead of single-node selection, enabling chain-of-thought through the graph.

\textbf{Task-specific node types}: Beyond generic ``insight'' nodes, support structured types (e.g., ``hypothesis'', ``evidence'', ``refutation'') with type-aware retrieval.

\textbf{Benchmark generalization}: Validate on standardized multi-agent benchmarks (GAIA Level 2, AgentBench) to demonstrate cross-domain applicability.

\section{Conclusion}

We presented SW-KG, a coordination architecture for multi-agent LLM systems that reduces token costs from $O(A \cdot T^2)$ to $O(A \cdot T \cdot \log^2 N)$ through small-world graph topology, hub compression, and token economy. Empirical validation on a 100-turn multi-agent task demonstrates 91.8\% token reduction with 12.19$\times$ speedup over baseline full-history approaches, significantly exceeding prior work (AgentDropout: 21.6\% reduction). Ablation studies confirm that all three mechanisms contribute substantively, with small-world topology providing the largest efficiency gain. Complexity curve fitting validates theoretical predictions with $R^2 = 0.9997$ for baseline quadratic scaling and $R^2 = 0.9232$ for SW-KG log-linear scaling.

Unlike task-specific optimizations, SW-KG operates as a domain-independent substrate applicable to any multi-agent system requiring cumulative knowledge sharing. The architecture achieves crossover at turn 10 (10\% of task length), after which efficiency gains compound asymptotically. For long-running collaborative tasks, SW-KG provides a practical path to scalable multi-agent coordination without sacrificing coherence.
